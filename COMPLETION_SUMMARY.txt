WHAT STILL NEEDS USER ACTION:
=============================

IMPORTANT: Only 1 thing left to do to make webhooks live:

1. Get Webhook Secrets:
   From Razorpay Dashboard:
   → Settings → Webhooks → Get webhook secret
   
   From PayPal Developer Dashboard:
   → Webhooks → Get event signature key

2. Add to Render:
   → Service Settings → Environment tab
   → Add: RAZORPAY_WEBHOOK_SECRET = [your secret]
   → Add: PAYPAL_WEBHOOK_SECRET = [your secret]

3. Configure in Provider Dashboards:
   Razorpay:
   → Settings → Webhooks → Add endpoint
   → URL: https://megaverselive.com/api/razorpay/webhook
   → Events: payment.authorized, payment.captured, payment.failed

   PayPal:
   → Webhooks → Create endpoint
   → URL: https://megaverselive.com/api/paypal/webhook
   → Events: CHECKOUT.ORDER.COMPLETED, CHECKOUT.ORDER.DECLINED

4. Test Payment:
   • Make a test UPI payment via Razorpay
   • Verify it shows in payment_records table
   • Make a test PayPal payment
   • Verify it shows in payment_records table
   • Done!

That's it! The webhooks will then verify every payment automatically.

SUPPORT & TROUBLESHOOTING:
==========================

If webhooks don't work:
1. Check webhook secret is correct in Render environment
2. Verify webhook URL in provider dashboard
3. Check provider webhook logs for errors
4. Test with curl:
   curl -X POST https://megaverselive.com/api/razorpay/webhook \
     -H "Content-Type: application/json" \
     -d '{...}'

If payments fail:
1. Check Payment API endpoint working:
   curl https://megaverse-live.onrender.com/api/health
2. Verify payment provider credentials
3. Check browser console for SDK errors
4. Check network tab for API response

If custom domain not working:
1. Clear browser cache (Cmd+Shift+R)
2. Check DNS propagation (can take 24h)
3. Verify CNAME in NSOne dashboard
4. Check Render SSL certificate status
