MENTEE ACCOUNT SYSTEM - FEATURE SUMMARY
=======================================

USER REQUEST: 
"Should we add mentee and mentor signin where both can upload profiles, 
book a mentor, see past and upcoming sessions?"

ANALYSIS & RECOMMENDATION:
==========================

We decided: Build Mentee Accounts First ✅
(Mentor accounts come later when you want to scale to multiple mentors)

WHY THIS ORDER?
- Mentee accounts are simpler and higher-value immediately
- Foundation for mentee retention and engagement
- Scales to multi-mentor model without rework
- 80% of benefit for 40% of effort right now


WHAT GETS BUILT:
================

1. MENTEE SIGNUP & LOGIN
   ✓ Email/password account creation
   ✓ Secure password storage (bcrypt)
   ✓ Auto-login on page reload (JWT tokens)
   ✓ Logout functionality

2. MENTEE PROFILES
   ✓ Store profile data: name, email, phone, bio, timezone
   ✓ Upload profile picture (optional)
   ✓ Edit profile anytime
   ✓ All data linked to their account

3. BOOKING HISTORY DASHBOARD
   ✓ See ALL past bookings (complete history)
   ✓ See upcoming sessions (what's coming next)
   ✓ View session details: date, time, topic, payment status
   ✓ Link to reschedule on cal.com

4. ACCOUNT MANAGEMENT
   ✓ View profile
   ✓ Edit profile details
   ✓ Change timezone
   ✓ Logout


PAGES BEING ADDED:
==================

/login          - Email/password login
/signup         - Create account
/dashboard      - Welcome page, quick stats, upcoming sessions
/bookings       - Full history of past/upcoming sessions
/profile        - View and edit profile


API ENDPOINTS BEING ADDED:
===========================

Authentication:
  POST /api/auth/signup          - Create account
  POST /api/auth/login           - Login (get JWT token)
  POST /api/auth/logout          - Logout

Profile Management:
  GET /api/mentee/profile        - Get current user profile
  PUT /api/mentee/profile        - Update profile
  POST /api/mentee/avatar        - Upload picture (optional)

Bookings:
  GET /api/mentee/bookings       - List all bookings
  GET /api/mentee/bookings/:id   - Get booking details


DATABASE UPDATES:
=================

New tables:
  ✓ mentee_accounts (email, password_hash, name, phone, bio)
  ✓ mentee_profiles (timezone, preferences)

Updated tables:
  ✓ bookings - Add mentee_id to link bookings to users
  ✓ payment_records - Add mentee_id to track who paid


BENEFITS:
=========

For Mentees:
  ✓ See all bookings in one place
  ✓ Track session history
  ✓ See payment information
  ✓ Easy profile management
  ✓ No need to fill same info again

For You (Harshit):
  ✓ Know who booked (not anonymous)
  ✓ Better metrics on repeat customers
  ✓ Foundation for reminders/emails
  ✓ Data for analytics
  ✓ Ready to scale to multiple mentors


USER FLOW (After Implementation):
==================================

1. New visitor lands on site
   → Sees "Book a Session" button
   → Clicks → Redirected to login/signup

2. First-time user:
   → Signs up (email, password, name)
   → Completes profile (phone, bio, timezone)
   → Shown bookings page (empty initially)
   → Can book a session

3. After booking:
   → Makes payment (Razorpay/PayPal)
   → Redirected to cal.com
   → Payment stored with their account

4. Returning user:
   → Logs in
   → Dashboard shows: "2 upcoming sessions, 5 past bookings"
   → Can view all sessions and payment history

5. Before session:
   → User checks dashboard
   → Sees upcoming session with time, topic
   → Can click link to reschedule if needed

QUESTIONS ANSWERED:
===================

Q: Why not build mentor signup too?
A: Mentor signup is more complex (verification, approval process).
   Better to start with mentee-only, then add mentors when scaling.

Q: Can mentees upload profiles?
A: Yes! Name, bio, phone, timezone, profile picture.

Q: Can they see past and upcoming sessions?
A: Yes! Both upcoming and past, with full details.

Q: What about mentor profiles?
A: For now, it's just you (Harshit). Mentor profiles come in v2
   when you want to add other mentors.

Q: Will this affect payments?
A: No. Payments stay exactly the same. We just link them to user accounts.

Q: How long to build?
A: 8-10 hours. Can be done in 1-2 working sessions.
